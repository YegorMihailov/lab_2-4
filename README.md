## Очередь задач TaskQueue

Реализована асинхронная очередь задач ( TaskQueue ) (пользовательская коллекция), поддерживающая итерацию, фильтрацию и потоковую обработку задач. Ленивые фильтры по статусу и приоритету

## Модель задачи Task

Реализована модель задачи ( Task ) в рамках платформы обработки задач с инкапсуляцией и валидацией состояния

## Атрибуты задачи

- идентификатор задачи (id)
- описание задачи (description)
- приоритет (priority)
- статус задачи (status)
- время создания (_created_at)
- готовность к выполнению (ready_to_start)

## Источники задач

- GeneratorTaskSource. Генерирует случайное количество задач (1-5)

- ApiTaskSource. Имитирует сетевое взаимодействие с задержкой в 1 сек. Возвращает фиксированный набор задач.

- FileTaskSource. Возвращает список задач из JSON-файла.

## Обработчики задач

- FileHandler. Генерирует случайное количество задач (1-5)

- LoggingHandler. Имитирует сетевое взаимодействие с задержкой в 1 сек. Возвращает фиксированный набор задач.

## Технические решения

- Контракт описан с помощью typing.Protocol. Классы GeneratorTaskSource, ApiTaskSource, FileTaskSource не наследуются от TaskSource

- Все источники задач реализуют единый контракт (метод
get_tasks())

- Реализована runtime-проверка соблюдения контракта в функции run_tasks;

- Архитектура позволяет добавлять новые источники без изменения
существующего кода

- Использование пользовательских дескрипторов для валидации атрибутов

- Использование property для вычисляемых и защищённых свойств

- Использование __slots__ в классе Task

- Корректная работа с большими объёмами задач с помощью yield

- Использование контекстных менеджеров для управления ресурсами

- Централизованное логирование и обработка ошибок
